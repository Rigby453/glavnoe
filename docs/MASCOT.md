# Kaizen — Маскот «Kai» (фирменная сущность)

> Источник истины по маскоту. Связан со SPEC §B4 (фирменные элементы), §B6 (тон),
> design-tokens.json (цвета тем), ANIMATIONS.md (длительности/кривые).
> Решение зафиксировано в decisions.md → **ADR-032**.
>
> **Имя `Kai` — рабочий кодовый идентификатор** (от *Kaizen*). Используется в коде,
> ассетах и доках. UI-имя/нейминг продукта может смениться позже — это замена одной
> строкой, на архитектуру не влияет.

---

## 1. Что это и почему

Не питомец, не животное, не робот, не человечек. **«Присутствие ИИ»** — фирменная
сущность приложения, которая воплощает собой продукт: «Главное само пересобрало день».

Целевая аудитория — взрослые (18–35, студенты). Поэтому образ ближе к **J.A.R.V.I.S. /
Nothing OS / Arc / Linear**, а не к Duolingo/Tamagotchi. Маскот с характером, но без
детскости — иначе взрослый пользователь начнёт прятать приложение.

**Эмоциональный микс:** ~70% J.A.R.V.I.S. (аналитическая холодность, «умный напарник») +
~20% Nothing OS (минимализм, премиальность) + ~10% Duolingo (привязчивость, характер,
дозированная пассивная агрессия в жёстком тоне).

## 1A. Роль: Kai — лицо и голос приложения (главный носитель ценности)

> Зафиксировано пользователем 2026-06-19. Kai — НЕ декоративный маскот в углу. Он
> **олицетворяет продукт и его ключевую механику**. «Просто увеличить размер» —
> недостаточно: Kai должен **играть значимую роль**, а не украшать экран.

**Принцип:** всё, что приложение «делает за пользователя», подаётся через Kai. Он —
лицо (визуальное присутствие на каждом экране) и голос (короткие реплики через
`KaiSpeechBubble`, тон gentle/harsh).

1. **Kai ведёт пересборку дня (главный хук продукта).** Утренний и вечерний разборы
   подаёт Kai: «Перенёс 2 несделанных на завтра, экзамен оставил защищённым — ок?» →
   пользователь жмёт «Принять». Авто-перераспределение **олицетворяет Kai**, а не
   безличная карточка.
2. **Kai объясняет «почему» (дифференциатор «понять, почему планы срываются»).** Не просто
   переносит задачи, а даёт инсайт: «Третий вечер подряд срываешь поздние задачи —
   перенести их на утро?». Это превращает Kai из «милого» в **полезного**.
3. **Реагирует на реальное состояние** (эмоция + реплика): просрочка → anxious + наджик;
   всё главное закрыто → success + похвала; давно не заходил → away + «вернись, соберём
   день». (Эмоции — см. §5; away уже определён, должен триггериться на пустом/забытом дне.)
4. **Тап по Kai → мини-чек-ин / быстрые действия:** «Как настрой?» (кормит дневник) или
   быстрые действия дня. Kai — точка входа, а не статичная картинка.
5. **Контекстное присутствие на каждом экране** (с уместной эмоцией и репликой): Today
   (шапка, реакция на прогресс), разбор (ведёт), celebration (гордость; в harsh —
   сдержанно), Food («подберём меню под твою норму»), Focus (в наушниках, «не отвлекайся»;
   при попытке свернуть — реагирует), загрузка (`KaiLoader` — думает вместо спиннера),
   пустые состояния, онбординг (эмоция на каждом шаге), виджет.
6. **Личность растёт от стрика:** чем стабильнее пользователь ходит — тем живее Kai;
   при срыве — сдержанная обеспокоенность. Лёгкая привязанность без детскости.
7. **Тон gentle/harsh** влияет на формулировки и пластику (см. §4) — реплики Kai всегда
   в выбранном тоне и на языке пользователя (l10n).

**Технически:** текущая реализация — чистый Flutter `CustomPainter` (6 эмоций + речь +
контекст) — уже покрывает пункты 1–5 без Rive. **Rive** (см. §7) даст плавную мимику и
сложные переходы, но требует ассета от дизайнера — это отдельный шаг (контент пользователя).

**Что это значит для реализации (бэклог):** Kai-ведомый разбор (Kai подаёт план, кнопка
«Принять») · строка «почему» из rule-based/AI-инсайта · тап-чек-ин настроения · контекстные
реплики на Food/Focus/пустых экранах · триггер эмоции `away` · harsh-вариант celebration.
Off-toggle в Профиле сохраняется (Kai можно отключить — деградирует в нейтральный UI).

## 2. Форма

- **База — мягкий squircle** (скруглённый прямоугольник, «тёплая галька»). **Не круг**
  (клише голосовых ассистентов — Siri/Google), **не квадрат** (корпоративно). Squircle
  перекликается с геометрией карточек и иконки приложения → сущность выглядит как
  «материя интерфейса», родная экрану.
- **Только «голова», без тела** — без рук, ног, одежды. Руки/ноги = территория Duolingo.
  (Опция «голова + маленький стаб-тело» НЕ реализуется в v1; добавляется одним компонентом
  позже, если базовый образ покажется холодноватым.)
- **Только два глаза-тире. Рта нет.** Рот = мультяшность. Весь эмоциональный диапазон —
  через **форму и цвет глаз** + пластику тела.
- **Асимметрия глаз: левый на ~1.5 px выше правого.** Это даёт характер без детскости.
  Симметричное лицо читается как иконка, асимметричное — как персонаж.

### 2.1 Морфинг (squircle, который «течёт» точечно)

База держит форму **в ~95% времени** (в виджете, в списках, в покое — стабильный,
читаемый). На **ключевых эмоциональных моментах** squircle деформируется — это даёт
премиальную «текучесть» liquid-концепта, но точечно, без постоянной анимации (дёшево +
читается мелко). В Rive это один параметр морфинга между заданными крайними формами.

| Состояние | Морф формы |
|---|---|
| Покой (idle) | ровный squircle, едва заметно «дышит» (масштаб ±2%) |
| Думает / ИИ строит план | чуть вытягивается по вертикали, мягко пульсирует |
| Радость / всё главное закрыто | расплывается почти в круг, пружинит |
| Испуг / тревога (свернул фокус, горит дедлайн) | резко сжимается, углы заостряются, дёрг |
| Провал / потеря стрика | «оплывает» вниз, нижний край растекается по границе карточки |

## 3. Глаза = единственный цветной элемент (маркер темы)

Тело — нейтральное (в тон поверхности темы). **Цвет глаз = акцент активной темы.** Это
делает Kai автоматически фирменным под каждую тему, без отдельных ассетов.

| Тема | Цвет глаз (= accent) | Тело |
|---|---|---|
| Focus | `#D9F24B` (лайм) | тёмно-угольное (в тон `#241D11`) |
| Calm | `#6FB6A3` (сине-зелёный) | полупрозрачное матовое, мягкое свечение |
| Black | `#C8FF4D` | чёрное (OLED), максимальный контраст |
| White | `#5B7CFA` (синий акцент) | тёмное, как капля туши (контраст на белом) |
| Contrast | accent темы | сплошное, max-контраст, **морфинг минимальный** (доступность) |

В **жёстком тоне** цвет глаз уходит в **ember** (`#FF6A3D` в Focus; в остальных темах —
«срочный/негативный» цвет темы). См. §4.

## 4. Два характера = существующий тумблер тона (gentle / harsh)

**Новой настройки НЕ вводим.** Маскот считывает уже существующий `tone` из prefs
(SPEC §B6, app/CLAUDE.md «Tone: gentle/harsh … affects display strings ONLY»). Тон меняет
поведение и форму Kai — пользователь сам выбрал, значит «бесячесть» легитимна.

| | Gentle (по умолчанию) — «J.A.R.V.I.S. с душой» | Harsh (опционально) — «Duolingo на максималках» |
|---|---|---|
| Форма | округлая, плавная, «дышит», левитация | резче; вытягивается в струну от напряжения, углы острее |
| Глаза | широкие точки / добрые полумесяцы `^ ^` | узкие щели `- -`, цвет → ember, «приподнятая бровь» (черта над глазом) |
| Поведение | помощник-дворецкий: элегантно подаёт задачи, мягко кивает | строгий надзиратель: смотрит на просрочку, потом на тебя, тяжело «вздыхает» (раздувается/сдувается) |

**Тексты Kai = существующие тон-строки из SPEC §B6.** Маскот их «произносит» (показывает
рядом с собой), но **отдельной копирайт-системы не заводим** — это те же строки, что уже
в карточках разбора. Без шейминга еды/тела (правило §B6).

## 5. Шесть выражений (форма + цвет глаз)

1. **Нейтральное** — глаза-тире, асимметрия, ровный squircle. База.
2. **Готово / успех** — арки глаз вверх (`^ ^`), форма пружинит к кругу.
3. **Обработка / думает** — один глаз прищурен, форма вытянута, лёгкая пульсация.
4. **Жёсткий режим** — глаза сплющены в плоские тире, цвет → ember `#FF6A3D`.
5. **Тревога** — глаза чуть крупнее, форма сжата/дёрганая.
6. **Давно не заходил** — глаза «нитки» (почти закрыты), форма слегка осевшая.

Тапабельно (в виджете/на Today) — для дебага/радости можно листать выражения.

---

## 6. Где живёт Kai (ambient, не добавляет действий)

Правило: **Kai никогда не перекрывает контент и не добавляет тапов.** Он фоновый,
тапабелен для смены настроения/дисмисса, целиком отключается тумблером в Профиле.

- **Today** — в шапке рядом с кольцом «главное X/Y»: реагирует на прогресс (idle → думает
  при разборе → радость на 100%).
- **Карточки разбора (утро/вечер)** — Kai «подаёт» задачи: при предложении переноса
  виртуально «двигает» блоки; он — тот, кто произносит тон-строку.
- **Виджет** (домашний экран) — маленький Kai + выражение по состоянию дня.
- **Celebration «всё закрыто»** — gentle: сальто/идеальный круг, лёгкое конфетти; harsh:
  сдержанный кивок, пиксельные очки «deal with it», растворяется.
- **Фокус-сессия (67/15 и др.)** — Kai в углу в наушниках, «дышит»; при попытке свернуть
  приложение — резко открывает глаза, снимает наушник, осуждающий взгляд (это и есть
  «трение при выходе» из SPEC §C8, без слежки за приложениями).
- **Empty states / «давно не заходил»** — выражение «нитки».
- **Health/Water (позже, Ф1-полиш)** — может плавать в «стакане», подпрыгивать на волне
  при отметке воды. Необязательно для первой версии маскота.

## 7. Реализация (для flutter-агента, когда дойдём)

- **Инструмент: Rive** (не Lottie). State machine: дизайнер рисует один раз, разработчик
  шлёт входы — Kai плавно перетекает между состояниями без тяжёлых видео.
- **Входы state machine (предложение, финализирует дизайнер):**
  - `isHarsh: bool` — из текущего тона (prefs).
  - `emotion: number` (0–5) — одно из шести выражений §5.
  - `morph: number` (0–1) — степень деформации формы §2.1.
  - `action: trigger` — разовые проигрыши (juggle / move-cards / celebrate / sigh).
  - `themeAccent: color` — цвет глаз = accent активной темы (§3).
- **Вес:** векторный Rive — килобайты; приложение остаётся быстрым.
- **Доступность / reduce-motion:** при `MediaQuery.disableAnimations` (см. ANIMATIONS.md
  §10) — Kai статичен (нейтральное выражение, без морфинга/циклов). В Contrast-теме морфинг
  минимален всегда.
- **Тумблер «Показывать Kai»** в Профиле (§C7) — часть взрослой аудитории маскотов не
  выносит; off = сущность скрыта, функционал не страдает.
- **Длительности** — по ANIMATIONS.md (snap 120 / fast 180 / normal 280 / slow 400 мс);
  фоновые циклы (дыхание) — медленные, зацикленные.

## 8. Что НЕ делаем (явные анти-паттерны)

- ❌ Полноценный человечек (руки/ноги/одежда) → Duolingo/Discord/Finch.
- ❌ Робот `🤖` → клише 2018-го.
- ❌ Супер-милые глаза `🥺` → взрослые начнут стыдиться приложения.
- ❌ Рот / речевые пузыри как у мультяшного персонажа.
- ❌ Постоянная тяжёлая анимация (видео/Lottie-комплекс) → вес и тормоза.
- ❌ Отдельная копирайт-система для Kai → переиспользуем тон-строки §B6.

## 9. Статус / фазирование

Это **дизайн-слой поверх готового функционала** (фичи MVP+Ф1+Ф2+Ф3 закрыты). Маскот не
блокирует и не меняет логику — он визуальный. Предлагаемый порядок (после утверждения
концепта и появления Rive-ассета от дизайнера):

1. Rive-ассет Kai (дизайнер) + 6 выражений + морф-состояния.
2. Интеграция на Today (шапка, реакция на прогресс/разбор) + тумблер в Профиле.
3. Виджет + celebration + фокус-сессия.
4. Health/Water-сценарии (Ф1-полиш, опционально).
