# Kaizen — Навигация и раскладка (UX по науке)

> Источник истины по расположению элементов управления: где что лежит, на какой акцент,
> как сократить число действий. Дополняет SPEC §B1 (принципы) и §C (экраны), не заменяет.
> Зафиксировано в decisions.md → **ADR-033**. Цвета — design-tokens.json. Маскот — MASCOT.md.
>
> **Легенда статусов:**
> - **[ОК — оставить]** — уже реализовано и соответствует науке; не трогать.
> - **[Шлифовка]** — предлагаемое уточнение поверх текущего; менять аккуратно.

---

## 1. Принципы (на чём основано)

| Закон / принцип | Что значит для нас |
|---|---|
| **Зона большого пальца** (Hoober: ~49% держат телефон одной рукой) | Частые и «позитивные» действия — внизу (лёгкая досягаемость). Редкие и опасные — вверху (трудная зона), специально. |
| **Закон Фиттса** | Главные цели — крупные и близкие к большому пальцу. FAB и таб-бар — крупные тач-цели (B1: «крупные тач-цели»). |
| **Закон Хика** | Чем меньше выбор, тем быстрее решение. 4 таба (не 5–6), max 3 «главных» задачи в день — уже в ТЗ. |
| **Закон Миллера (7±2)** | Не больше ~5 пунктов в одном ряду меню/секции без группировки. |
| **Закон Якоба** | Пользователь ждёт поведения как в знакомых приложениях (Things/TickTick/Todoist/Apple) — не изобретаем навигацию заново. |
| **Закон Теслера (сохранение сложности)** | Сложность нельзя убрать, только перенести. Переносим её с пользователя на приложение (умные дефолты, авторазбор, голос). |
| **Прогрессивное раскрытие** | Сначала простое (1 тап «Принять разбор»), детали — по запросу («Поправить»). |

**Сквозная цель (B1): 1–2 тапа на действие.** Каждое решение ниже работает на это.

## 2. Карта досягаемости (где что и почему)

```
┌─────────────────────────────────┐
│ [аватар]            [тон-тумблер]│  ← ВЕРХ: редкие/безопасные (трудная зона — намеренно)
│                                  │
│            КОНТЕНТ               │  ← скролл-зона: задачи/таймлайн/еда
│         (карточки, ряды)         │
│                                  │
│                        ╭──────╮  │  ← НИЗ-ПРАВО: главное создание (зона пальца)
│                        │+ Add ◓│  │     FAB «+ Add» с голосом
│                        ╰──────╯  │
│ [Today] [Plan] [Health] [Diary]  │  ← НИЗ: навигация (самая лёгкая зона)
└─────────────────────────────────┘
```

- **Профиль — аватар слева сверху** (не таб). **[ОК — оставить]** Это самая трудная для
  правого пальца зона — и это правильно: профиль/настройки заходят редко, нельзя тратить на
  них прайм-место. (SPEC §C7, app/CLAUDE.md.)
- **Тумблер тона — справа сверху.** **[ОК — оставить]** Тоже редкое действие, трудная зона.
- **Нижний таб-бар + FAB — внизу.** **[ОК — оставить]** Частое — под большим пальцем.

## 3. Нижняя навигация (таб-бар)

**4 таба: Today · Plan · Health · Diary.** **[ОК — оставить]** (SPEC §C, app/CLAUDE.md §3.)

- **Почему 4, а не 5:** закон Хика; профиль вынесен в аватар, чтобы не плодить пятый таб.
- **Иконка + подпись** у каждого таба — для первого узнавания (не только иконки).
- **Активный таб = акцент-«пилюля»** (лайм в Focus) — на скриншотах уже так («Today» в
  лаймовой пилюле). Это единственный цветной элемент таб-бара.
- **[Шлифовка] FAB не должен сливаться с таб-баром.** На скриншоте «+ Add» висит вплотную
  над навигацией — задать гарантированный зазор (≥16 dp) и тень/контраст, чтобы FAB читался
  как отдельный слой, а не часть бара. На узких экранах (360 px) проверить, что FAB не
  перекрывает подпись правого таба (Diary).

## 4. Главное создание — FAB «+ Add»

**[ОК — оставить]** FAB снизу-справа, с иконкой микрофона (голосовой ввод). Это сильный
дифференциатор: голос = создание задачи за 0 шагов ввода с клавиатуры.

- **[Шлифовка] Поведение при скролле:** extended-FAB («+ Add» с текстом) в покое →
  сворачивается в компактный иконочный FAB при скролле вниз (меньше шума над контентом),
  разворачивается при скролле вверх. Material-паттерн, знаком пользователю (закон Якоба).
- **[Шлифовка] Голос — всегда один тап.** Микрофон в FAB ведёт сразу в надиктовку (естеств.
  ввод уже в ТЗ §C2), не пряча его в подменю.
- FAB присутствует на **Today** и **Plan** (создание задач/событий). На **Health** главное
  действие — контекстное (лог еды/воды), не общий «+ Add». На **Diary** — «Сохранить день».

## 5. Расположение по экранам (привязка к скриншотам)

### Today **[ОК — оставить структуру]**
Сверху вниз: шапка (приветствие + дата, аватар слева, тон справа) → **кольцо «главное
X/Y» + стрик + 7 точек** → карточка разбора (если есть) → **«Главное сегодня» (1–3
защищённые)** → **«Дальше по дню» (хроно-таймлайн)** → FAB «+ Add».
- Порядок верный по приоритету внимания: сначала «сколько главного закрыто», потом само
  главное, потом остальное. **[Шлифовка]** Kai (MASCOT.md) живёт в шапке рядом с кольцом.

### Plan **[ОК — оставить]**
Верхний сегмент **Day / Week / Month** → недельная полоса (свайп) → дневной таймлайн по
времени → закреплённая карточка экзамена/дедлайна с обратным отсчётом (ember).
- Сегмент-переключатель сверху — знакомый паттерн (Fantastical/Apple Calendar, закон Якоба).
- **[Шлифовка]** «EXAM COMING» (ember-карточка) — закрепить вверху ленты, не уезжает при
  скролле, пока дедлайн актуален.

### Health (сводка + опциональные модули) **[ОБНОВЛЕНО под код]**
Не хаб-с-табами и не сетка из 10 плиток. Экран — **вертикальная сводка**: заголовок «Здоровье»
→ **карточка Вода** (лог + недельный мини-график) → **карточка Сон** (Ложусь/Проснулся + график)
→ **компактные плитки-входы только для включённых режимов-флагов** (Еда / Тренировки / Медитация /
Дыхание — см. SPEC §C5, §C8b).
- **Вода и Сон видны всегда**; тяжёлые модули — за флагами в Профиле → «Поведение». Если флаги
  выключены (дефолт) — экран = Вода + Сон, без плиток.
- Лог-действия (вода, сон) — **прямо на карточке** (Outlined-кнопки, тап = записал); глубже —
  по `chevron_right` в полный экран/отчёт. Главное действие здесь **контекстное**, не общий FAB.
- Осанка/привычки/со-учёба **из этого экрана убраны** (осанка — тумблер в Профиле; привычки
  удалены; со-учёба выведена из нав).
- Пофайловая раскладка элементов каждого экрана раздела — в **`/docs/UX-LAYOUT-FULL.md` §B–E**.

### Diary **[ОК — оставить]**
Настроение → план vs факт → «что пошло не так» (чипы) → инсайт → «Сохранить день».
- Главное действие — нижняя кнопка **[Сохранить день]**, не FAB.

## 6. Акцент и иерархия цвета (на какой акцент)

Один насыщенный цвет среди нейтралов работает **пред-внимательно** (глаз цепляется
мгновенно). Поэтому акцент — дефицитный ресурс, не «заливка».

**Правила [Шлифовка — кодифицировать]:**
- **Accent (лайм `#D9F24B` в Focus) = только:** (1) главное действие/FAB, (2) текущий выбор
  (активный таб/сегмент), (3) успех/готово (галочки, замыкание кольца), (4) маркер
  **единственно главного** (щит у protected-задачи). Глаза маскота — тоже accent (MASCOT.md).
- **Ember (`#FF6A3D`) = только срочное/негативное:** просрочка, обратный отсчёт экзамена,
  жёсткий тон. Никогда не смешивать с accent в одном элементе.
- **Нейтраль (текст/мутед/бордер)** — всё остальное.
- **[Шлифовка] «Стена лайма» в Food:** сейчас лаймом залиты и прогресс-бар, и все бары
  макро, и галочки. Предложение: акцентом — только заголовочная метрика (ккал к норме) и
  отметки «съел»; вторичные бары (Б/Ж/У) — приглушённый тон; «следящие» метрики (Сахар —
  красный, Клетчатка) — свой семантический цвет. Так акцент снова означает «важное», а не
  «фон».

## 7. Сокращение числа действий (закон Теслера на практике)

Сложность несёт приложение, не пользователь:

- **Авторазбор = batch-действие.** Утренний/вечерний разбор: **[Принять] одним тапом**
  применяет все переносы. **[ОК]** «Поправить» — прогрессивное раскрытие для тех, кому надо.
- **Умные дефолты при создании. [Шлифовка]** Новая задача предзаполнена: дата = сегодня,
  время = ближайший свободный слот, приоритет = обычный. Пользователь часто просто жмёт
  «Сохранить». Естественный ввод парсит «завтра 17:00» из текста (ТЗ §C2).
- **Голос. [ОК]** Микрофон в FAB и в поиске еды — ввод без клавиатуры.
- **Свайпы по задаче (готово/пропуск). [ОК]** — но **[Шлифовка]** нужна видимая подсказка
  аффорданса при первом использовании (свайпы не самоочевидны — закон открываемости).
- **Однотапные логи. [ОК]** «Съел по плану», тап по стакану = выпил.
- **Клонирование недели. [ОК]** Импорт расписания одним действием.

## 8. Что заимствуем у популярных планнеров (закон Якоба)

| Источник | Паттерн | Где у нас |
|---|---|---|
| **Things 3** | Воздух/иерархия; «Today» как дом; один ясный главный экран | Today как стартовый таб, минимум хрома |
| **TickTick / Todoist** | Нижний таб-бар + FAB; естественно-языковой quick-add | FAB «+ Add», парсинг «завтра 17:00» |
| **Sunsama / Structured** | Тайм-боксинг дня вертикальным таймлайном; спокойный ритм | «Дальше по дню» (Today) и лента Plan |
| **Fantastical / Apple Calendar** | Сегмент День/Неделя/Месяц | Переключатель на Plan |
| **Apple Reminders** | Секции CAPS-мутед-заголовками | «REST OF THE DAY», «MEALS» — уже так |

Не копируем буквально — берём знакомые расположения, чтобы не заставлять переучиваться.
