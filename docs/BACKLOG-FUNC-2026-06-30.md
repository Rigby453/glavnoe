## Аудит реализации (2026-06-30, read-only) — текущее состояние [INV]-пунктов
> Сделан аудит кода. Ниже — что есть и что доработать. Реализацию делать с проверкой на устройстве.

- **C3 Logout/смена аккаунта — ЕСТЬ, но с дырой.** `auth_controller.logout()` + кнопка в Профиле
  (`profile_screen.dart:360`) очищают токен и редиректят на /auth. **ПРОБЕЛ (важно):** НЕ чистятся
  локальные Drift-данные → при входе в ДРУГОЙ аккаунт видны задачи предыдущего пользователя.
  Доработать: очистка items/streaks/water_logs/day_logs при logout. Смена аккаунта = только через
  полный re-login.
- **C2 Гостевой режим — ЕСТЬ** (кнопка «Continue as guest» на auth, `continueOffline()` + флаг
  `guest_mode`; данные пишутся в Drift, sync off). **ПРОБЕЛЫ:** (1) при login гостя его локальные
  данные НЕ мигрируют на сервер (`_clearGuest()` только снимает флаг) → риск потери; нужна миграция
  гостевых данных на первый sync. (2) онбординг не предлагает «пропустить» → гость.
- **B3 Повторяющиеся задачи — ЕСТЬ базово.** Якорь с RRULE (EXDATE/UNTIL) + виртуальные повторы
  (`recurrence.dart`, `items_dao.dart`): удалить одну дату (EXDATE), остановить серию (UNTIL,
  `stopSeries`), удалить всю серию (`deleteItem` якоря). **ПРОБЕЛ = B4:** при ПЕРЕНОСЕ повтора нет
  выбора «эта / все / будущие» (как в Google Calendar) — переносится только конкретный день. Нужно
  меню выбора при drag/правке времени + UPDATE соответствующих строк. (Минор: EXDATE копится текстом.)
- **F1 Секундомер в фокусе — НЕТ.** `focus_screen.dart` имеет таймер ОБРАТНОГО отсчёта (5 пресетов
  25/5…67/15, Timer.periodic, счётчик блоков), но НЕТ режима СЕКУНДОМЕРА (count-up). Доработать:
  toggle count-up/down или отдельный режим. (Также нет звука окончания фазы и persist при закрытии.)
- **D1 Уведомления — ЧАСТИЧНО, КРИТИЧНЫЕ дыры (это и есть «боль аналогов»).** Архитектура верная
  (zonedSchedule, таймзоны, runtime-permission), НО:
  1. 🔴 Везде `inexactAllowWhileIdle` вместо `exactAllowWhileIdle` → Android Doze откладывает/глотает
     (утреннее 8:00 может прийти в 10:30 или не прийти). 2. 🔴 В AndroidManifest НЕТ
     `SCHEDULE_EXACT_ALARM`/`USE_EXACT_ALARM` (есть только POST_NOTIFICATIONS). 3. ⚠️ Нет проверки,
     что канал не выключен пользователем. 4. ⚠️ Task-reminder НЕ перепланируются при смене таймзоны
     (TODO в коде). 5. ⚠️ Нет fallback (AlarmManager) при сбросе очереди ОС.
  **План доработки D1 (приоритет №1):** добавить SCHEDULE_EXACT_ALARM + USE_EXACT_ALARM в манифест;
  перевести critical-уведомления (daily reviews, task reminders) на `exactAllowWhileIdle`; добавить
  флоу запроса exact-alarm разрешения (Android 12+); проверять активность канала; перепланировать
  task-reminders в `refreshTimezone()`; гайд пользователю по разрешениям/battery-optimization.
  ⚠️ Требует проверки на реальном устройстве (exact-alarm бросает PlatformException без разрешения).
